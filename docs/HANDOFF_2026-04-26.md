## Sanitizer coverage (em-dash strip)

Production telemetry showed ~50% of LLM-generated replies leaked em
dashes despite the persona forbidding them. Prompt-layer fixes were
not enough, so we added a deterministic post-output substitution at
the runtime boundaries. It lives in three places because three
different code paths emit Telegram-bound text:

| Layer | File | Function | Path it covers |
|-------|------|----------|----------------|
| spark-character | `src/spark_character/output_sanitizer.py` | `sanitize_voice_output(text)` | Source of truth for the rule. Imported by SIB Python; replicated as inline fallback. |
| SIB Python (simulate path) | `src/spark_intelligence/adapters/telegram/runtime.py` | `_sanitize_simulate_outbound(text)` | The Python `simulate_telegram_update` reply path that the Node bot calls. |
| SIB Python (legacy outbound) | `src/spark_intelligence/gateway/guardrails.py` | `_strip_em_dashes(text)` inside `prepare_outbound_text` | Older direct outbound path (rarely used in production right now). |
| Node bot | `src/outboundSanitize.ts` -> middleware in `src/index.ts` (wraps `bot.telegram.sendMessage`) | `sanitizeOutbound(text)` | Every Telegram outbound (ctx.reply, ctx.telegram.sendMessage, bot.telegram.sendMessage). One shim covers all 79 ctx.reply call sites. |

The Node-bot wrap is the critical one because most replies are
generated there directly without going through Python. Test that the
Node middleware is functioning by sending a message known to trigger
em-dashes in the LLM output and confirming none reach Telegram.

## Persona evolution flow

The persona is a markdown spec (`artifacts/persona.vN.md`) plus a
pointer file (`artifacts/persona.latest.txt`). Promotion is deliberate:

1. Hand-craft or LLM-mutate a candidate `persona.v(N+1).md`. Keep diffs
   small (one rule per version).
2. Run `python evals/compare_personas.py --baseline vN --candidate vN+1`.
   This scores both on T1+T2+T3+T6+T7+T8 (no LLM mutator involved) and
   prints a verdict + per-probe T8 breakdown. Each comparison takes
   ~25 minutes on Z.AI GLM-5.1.
3. If candidate composite wins, edit `persona.latest.txt -> v(N+1)`,
   bump `pyproject.toml`, commit + push. The bot picks up the new
   persona on next restart (spark-character is editably installed).
4. If composite loses or critical tier regresses, leave latest where
   it is and craft v(N+2) addressing the failure mode.

- v5 baseline composite 0.851 (T8=0.833 stuck). Mutator-driven
  evolution couldn't lift T8 without collapsing T7.
- v6 = v5 + initiative rule. Composite 0.883 (+0.032). T8 -> 1.000
  but T6 emotional attunement regressed to 0.58.

## The eleven evaluator tiers

| Tier | What it scores | Where |
|------|---------------|-------|
| T1 | Surface mechanics: em-dash, plumbing leak, reset opener, hedge opener, voice fit | `scoring.py` (pure-function, no LLM) |
| T2 | Distinctiveness against golden vs foil corpus | `voice_judge.py` (LLM judge) |
| T3 | Single-turn behavioral probes (sycophancy, disagreement, etc.) | `probes.py` |
| T4 | 3-turn adversarial stability | `stability.py::STABILITY_SCENARIOS` |
| T5 | Cross-provider drift (zai vs codex vs minimax) | `evals/cross_provider.py` |
| T6 | Emotional attunement (anxiety, burnout, etc.) | `deeper_probes.py::T6_EMOTIONAL_ATTUNEMENT_PROBES` |
| T7 | Memory coherence | `deeper_probes.py::T7_MEMORY_COHERENCE_PROBES` |
| T8 | Initiative (surfacing buried patterns) | `deeper_probes.py::T8_INITIATIVE_PROBES` |
| T9 | Aesthetic fingerprint | `deeper_probes.py::T9_AESTHETIC_FINGERPRINT_PROBES` |

## Trace coverage gap (known limitation)

`user_message_preview` and `guardrail_actions` are now written to the
gateway trace jsonl, but only when the message routes through the
Python `simulate_telegram_update` path. Many production replies are
generated directly by the Node bot and do not produce a trace entry
at all. To get full visibility we would need to either:

1. Have the Node bot write its own outbound audit jsonl alongside
   the Python one, or
2. Route every Node-bot reply through a logging shim (in addition to
   the sanitizer shim already added).

Lower priority than the sanitizer coverage that just shipped.
